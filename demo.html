How to build a Microsoft 365 declarative agent · 8–12 minutes

What the demo covers: generating a system instruction using AI · opening M365 Copilot and finding the agent builder · configuring the agent with name, instruction, and connected files · adding starter prompts · testing with a real brief · comparing output to a blank Copilot session · sharing with a team.

Written guide — follow along

Build your agent — eight steps.

Follow these steps in order. Have your system instruction from Step 1 ready before you start.

1

Use AI to generate your system instruction

If you have not already done this, go back to Step 1 and run the prompt in Claude, ChatGPT, or M365 Copilot. Answer the eight questions. Copy the full instruction the AI generates and save it to a Word document or notes app. Do not summarise it — copy it in full.

2

Open Microsoft 365 Copilot

Go to microsoft365.com and sign in with your Microsoft account. Alternatively open Microsoft Teams and look for Copilot in the left sidebar. Navigate to the Copilot section — look for the agent or assistant builder. If you cannot find it, use the search bar inside M365 and type "agent builder."

3

Create a new agent

Click Create a new agent or New agent. You will see a configuration screen with fields for name, description, and instructions. Fill in the Name — be specific, for example: Quarterly Report Presentation Builder — Clarity Accounting. Add a one-sentence Description — what the agent does and who it serves.

4

Paste your system instruction

Paste your full system instruction from Step 1 into the Instructions field. Do not truncate or summarise it. The character count will be high — that is intentional. Detail is what makes this agent significantly more useful than a blank Copilot session. Review it once after pasting to confirm it pasted correctly.

5

Connect your knowledge files

Navigate to the Knowledge section and upload any relevant documents. These become the agent's reference library — it draws on them when generating presentation content. Useful files to upload include: report templates, policy documents, curriculum standards, product catalogues, financial summaries, brand guidelines, or any document your presentations typically reference. The more relevant the files, the more accurate and specific the output.

File types that work: Word documents (.docx), PDFs, Excel spreadsheets (.xlsx), and plain text files. Keep each file under 50MB. Use clear, descriptive file names so the agent can reference them accurately.

6

Add starter prompts

Find the Starter prompts or Conversation starters field and add three to five meaningful prompts. These appear as clickable suggestions when someone opens the agent. Make them specific and immediately useful. Examples for a financial reporting agent:

  • "Create a quarterly financial report presentation for Q3 FY2025 — revenue $487K, expenses $299K, ATO GST due 28 April."
  • "Draft a year-end financial summary presentation for a small business client — non-financial audience."
  • "Create a cash flow update presentation — business has $143K in bank, $67K outstanding debtors, payment due next month."
7

Test with a real brief

Use the test panel or start a conversation with your agent. Give it a real presentation brief from your work context. Evaluate the output against three questions:

  • Does the output reflect your professional context — or does it sound generic?
  • Does every slide title state the message as a complete sentence?
  • Is the tone and language appropriate for your stated audience?

If any answer is no — note what needs adjusting and refine your system instruction before deploying. Minor refinements are normal and expected on the first test.

8

Deploy to your team

Once you are satisfied with the output, share the agent with colleagues through the sharing or publishing settings in M365. In a Microsoft 365 tenant, you can publish the agent so it appears in Microsoft Teams for everyone in your organisation with access. One person builds it. Everyone benefits. Update the system instruction once — every team member gets the improvement immediately.

Save your system instruction. Copy it to a Word document and name it: M365 Agent Instruction — [Your Organisation] — June 2026. When you open a new session or need to reconfigure the agent, you have it ready to paste immediately.
