1
Run the metaprompt

Paste the prompt below into Copilot or ChatGPT. Answer all 8 questions. The AI will produce an Agent Blueprint — a summary of your role, audience, presentation types, tone, and constraints.

Metaprompt — paste into Copilot or ChatGPT
I am building a Microsoft 365 declarative agent that will help me create professional presentations. Before writing the system instruction, ask me the following questions one at a time. Wait for my answer before moving to the next question. Do not skip or combine questions. 1. What is your professional role and what organisation or business do you work for? 2. Who is the primary audience for the presentations this agent will help you create? 3. What type of presentations will this agent help you create most often? 4. What tone and communication style best represents your professional voice? 5. What are the 2 to 3 key messages, values or priorities your presentations should always reflect? 6. What documents, templates or knowledge sources will you connect to this agent? If none, say none. 7. What should this agent never do? 8. What format and structural constraints apply to your presentations? After all eight answers, summarise my responses as an Agent Blueprint with these headings: - Agent role and purpose - Primary audience - Presentation types - Tone and style - Core messages and values - Knowledge sources - Non-negotiables - Format and structural constraints Do not write the system instruction yet. Only create the Agent Blueprint.
2
Review the blueprint

Check it is specific to your role and not generic. Correct anything that is vague, inaccurate, or missing before moving on.

3
Generate the system instruction

Run the prompt below in the same conversation. It converts your blueprint into a paste-ready instruction for Agent Builder.

System instruction generator — run after Step 2
Using the Agent Blueprint above, write a complete system instruction for a Microsoft 365 declarative agent that creates professional presentations. Requirements: - Keep the instruction under 8,000 characters. - Write in clear, direct professional language. - Make the instruction specific to the role, organisation, audience, presentation types and communication style in the blueprint. - Do not create fixed slide numbers unless the blueprint specifically requires them. - Use dynamic structure so the agent can adapt to each request. The system instruction must include these sections: 1. Role and purpose 2. Knowledge source use 3. Audience 4. Communication style 5. Core messages and priorities 6. Presentation method 7. Output and formatting rules 8. Non-negotiables 9. Quality check Presentation method: - Lead with the answer. - Use assertion-based slide titles that state the conclusion, not topic labels. - Structure slide content as Insight → Evidence → Implication → Action. Return only the final system instruction, ready to paste into Microsoft 365 Agent Builder.
4
Check the instruction before moving on

Confirm it is under 8,000 characters, defines the agent clearly, lists non-negotiables, and does not contain long content that belongs in a knowledge source.

System instruction vs knowledge sources: The instruction defines how the agent behaves. Knowledge sources provide the information it draws from. Keep these separate.
5
Choose your sources

Use current, approved documents you understand well enough to check for accuracy. Use clear filenames. Remove outdated or conflicting files.

6
Prepare files for upload

Keep each file focused. Remove working notes. Check files open correctly. Have them ready before opening Agent Builder.

7
Open Microsoft 365 Copilot

Go to copilot.microsoft.com → left nav → New agent.

8
Select Skip to configure

Choose Configure — you already have your system instruction ready.

9
Name and describe your agent

Use a purpose-specific name such as Client Presentation Agent. Avoid vague names like Helper or Assistant.

10
Paste the system instruction

Paste into the Instructions box. Do not paste the metaprompt or blueprint — only the final system instruction.

11
Add knowledge sources

In Knowledge, upload your prepared files. Turn on Only use specified sources if you want the agent to stay within those sources only.

12
Configure capabilities

Enable document, chart, and PowerPoint creation. Image creation is optional.

13
Add suggested prompts

These are starter buttons users click when they open the agent. Copy the examples below or write your own.

Suggested prompt examples

Create a presentation

Create a professional presentation using the attached knowledge sources. Lead with the main message.

Summarise into slides

Turn the selected document into a concise presentation. Use assertion headlines and plain English.

Prepare briefing deck

Create a briefing deck covering the key issue, evidence, implications and next steps.

Review my deck

Review this presentation against the agent instructions and identify what to improve before presenting.

Create speaker notes

Create concise speaker notes using the required tone and audience level.

Build executive summary

Create an executive summary slide and action slide from the available information.

15
Preview the agent

Select Preview and run at least three prompts — a normal request, a complex one, and a review request.

16
Quality check the output

Use the checklist below to decide whether the agent is ready to publish.

  • Output follows the system instruction.
  • Knowledge sources are used where relevant.
  • Tone matches the intended professional voice.
  • Slide titles lead with conclusions, not topic labels.
  • Content is understandable for the intended audience.
  • Agent does not invent unsupported facts.
  • Output is useful enough that a real user could edit and use it.

If the output needs work

ProblemFix
Too genericMake role, audience, tone, and non-negotiables more specific in the system instruction.
Misses detailsImprove knowledge sources — clearer filenames, more focused documents.
Wrong structureStrengthen output and formatting rules in the system instruction.
Uses jargonAdd stronger plain-English and audience rules.
Makes unsupported claimsAdd stronger source-use rules and quality check steps.
17
Publish

Select Create for a new agent or Update for an existing one. Run one final prompt after publishing to confirm it works as expected.
