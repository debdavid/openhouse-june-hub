  • The steps are roughly the same each time
  • It uses information that can be written down or uploaded
  • Quality varies depending on how much time I have
  • I spend more time compiling or formatting than thinking

Part 2 — Design your digital colleague

Work through sections 1–7 in order. Vague inputs produce vague outputs — push for specificity in every section.

1
Agent name & who uses it

Name it specifically — not "My Agent." Example: "Never Miss a Deadline." Include your role and how often you would use it.

2
Current process & pain points

Walk through your current steps from start to finish. Where is it slow, inconsistent, or frustrating? What happens when you are rushed?

3
Decision points

What judgments do you make manually when doing this task? Example: Which emails are urgent? Which can wait? Which need escalation? These are what your agent will help structure.

4
Knowledge sources

List every source the agent needs — Outlook, Teams, SharePoint, uploaded templates, policy documents, past examples. If it is not listed here, the agent will not know to use it.

5
What the agent should produce

Describe the output specifically — sections, structure, length, tone. Example: a three-section daily briefing with Priority Action, This Week, and FYI/File.

6
Non-negotiables — what the human still decides

What professional judgment must stay with you? What must this agent never do? What always requires your review before it is used or sent?

7
Success looks like...

One or two sentences. How will you know this agent is working? Time saved? Consistent quality? Fewer missed commitments?

Four things that improve every agent interaction

Lead with the outcome

Not: "Create a progress note." Instead: "Create an NDIS progress note for [participant] working on Goal 2. Observations: [notes]. Three paragraphs, plain English, no jargon."

Name the audience every time

Even with audience context in your system instruction, naming the specific audience in each request sharpens the output. "Audience: family members — warm, plain English, no clinical terminology."

Give the most important thing first

State the key finding upfront. "The key finding is a 22% increase in response time. Build the status update narrative around what this means for the timeline."

Use the quality check as a prompt

"Review this output. Does it reflect my professional context or feel generic? Is there anything a professional in my field would find inaccurate? List what to fix, then rewrite."

Quality check — five questions before using any output professionally

QuestionWhat to check
Does it serve the stated aim?If you removed a section, would the audience miss anything essential? If not, cut it.
Does the structure make sense?Does it build logically toward the conclusion? Would a sceptical reader find it convincing?
Is the language right for this audience?Read one section aloud as if you are the least familiar person in the room. Is it still clear?
Are there any risks?Unverified claims, legally sensitive content, anything that could mislead. The agent does not know your legal obligations — you do.
Does it meet your professional standard?Consistent format, appropriate length, correct terminology. The agent applies the standards you described — verify they were followed.
